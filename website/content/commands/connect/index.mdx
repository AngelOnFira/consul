# Consul Connect

Command: `consul connect`

The `connect` command is used to interact with Consul's service mesh capabilities
managed by the connect subsystem. It exposes commands for:
- launching a sidecar proxy to bring mesh-unaware applications into the mesh
- viewing/updating the service mesh's Certificate Authority (CA) configuration

This command is available in Consul 1.2 and later.

## Usage

Usage: `consul connect <subcommand>`

For the exact documentation for your Consul version, run `consul connect -h` to view
the complete list of subcommands.

```text
Usage: consul connect <subcommand> [options] [args]

This command is used to interact with the service mesh's Certificate Authority
managed by the connect subsystem, including viewing or modifying the current CA configuration.
See the [service mesh CA documentation](/docs/connect/ca) for more information.

  This command has subcommands for interacting with the service mesh managed
  by the connect subsystem.

  Here are some simple examples, and more detailed examples are available
  in the subcommands or the documentation.

  Run and Configure an Envoy sidecar proxy for an instance of the web service:

      $ consul connect envoy -sidecar-for web

  For more examples, ask for subcommand help or view the documentation.

Subcommands:
    ca                  Interact with the Consul Connect Certificate Authority (CA)
    envoy               Runs or Configures Envoy as a service mesh proxy
    expose              Expose a service in the mesh through an Ingress gateway
    proxy               Runs a built-in service mesh proxy (not recommended for production)
    redirect-traffic    Applies iptables rules for traffic redirection
```

For more information, examples, and usage about a subcommand, click on the name
of the subcommand in the sidebar.
